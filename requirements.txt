# Light Polarization Simulator

An interactive educational repository for understanding **polarization of light** using two orthogonal sinusoidal electric-field components.

The simulator lets you control:

- the amplitude of the x-component \(A_x\);
- the amplitude of the y-component \(A_y\);
- the phase difference \(\delta\) between the two sinusoids;
- the wavelength \(\lambda\) of the propagating wave;
- the animation speed and trajectory persistence.

It then visualizes the resulting light polarization in four complementary ways:

1. **Face view (xy):** the polarization trajectory or polarization ellipse.
2. **Side view (xz):** the x-component of the wave while propagating along \(z\).
3. **Top view (yz):** the y-component of the wave while propagating along \(z\).
4. **Time traces:** the two component sinusoids \(E_x(t)\) and \(E_y(t)\).

![Preview of the polarization simulator](figures/polarization_preview.png)

## Why this repository?

The polarization state of a monochromatic electromagnetic wave is completely determined by the relationship between two orthogonal field components:

\[
E_x(t,z)=A_x\cos(kz-\omega t),
\qquad
E_y(t,z)=A_y\cos(kz-\omega t+\delta).
\]

By varying only the amplitude ratio \(A_y/A_x\) and the phase difference \(\delta\), one can generate:

- **linear polarization**;
- **circular polarization**;
- **elliptical polarization**.

This repository is designed to make those relationships intuitive and interactive.

## Live simulation

Open `index.html` directly in a browser, or publish the repository with GitHub Pages.

The simulator is self-contained in HTML, CSS, and JavaScript, and does not require a Python server.

### GitHub Pages publishing

1. Create a public GitHub repository named `light-polarization-simulator`.
2. Upload all files from this repository.
3. Open **Settings → Pages**.
4. Under **Build and deployment**, choose **GitHub Actions**.
5. The included workflow will publish the simulation automatically.

The final website will typically be:

```text
https://YOUR-GITHUB-USERNAME.github.io/light-polarization-simulator/
```

## What the simulation teaches

### 1. Linear polarization

If the two components are in phase or out of phase:

\[
\delta = 0 \quad \text{or} \quad \pi,
\]

the electric-field tip traces a straight line in the face view.

Examples:

- \(A_x=1,\ A_y=0\): linear polarization along x;
- \(A_x=0,\ A_y=1\): linear polarization along y;
- \(A_x=A_y,\ \delta=0\): linear polarization at \(+45^\circ\);
- \(A_x=A_y,\ \delta=\pi\): linear polarization at \(-45^\circ\).

### 2. Circular polarization

If the amplitudes are equal and the phase difference is \(\pm \pi/2\):

\[
A_x=A_y,
\qquad
\delta=\pm \frac{\pi}{2},
\]

the field tip traces a circle.

The sign of \(\delta\) determines the handedness.

### 3. Elliptical polarization

All other non-degenerate combinations produce an ellipse. Circular and linear polarization are therefore special cases of elliptical polarization.

## Stokes parameters

For ideal orthogonal harmonic components, the Stokes parameters are:

\[
S_0 = A_x^2 + A_y^2,
\qquad
S_1 = A_x^2 - A_y^2,
\]

\[
S_2 = 2A_xA_y\cos\delta,
\qquad
S_3 = 2A_xA_y\sin\delta.
\]

The simulator displays the normalized values:

\[
s_1 = \frac{S_1}{S_0},\quad
s_2 = \frac{S_2}{S_0},\quad
s_3 = \frac{S_3}{S_0}.
\]

It also computes:

- the **orientation angle** \(\psi\),
- the **ellipticity angle** \(\chi\),
- the **handedness**.

## Using the simulator

### Browser version

Simply open:

```text
index.html
```

in a browser.

### Python version

Install dependencies:

```bash
python -m pip install -r requirements.txt
```

Run the script:

```bash
python python/light_polarization_simulation.py --show
```

Save static figures:

```bash
python python/light_polarization_simulation.py \
  --ax 1.0 \
  --ay 0.7 \
  --phase 60 \
  --save-prefix figures/example_state
```
